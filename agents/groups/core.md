# 核心部组级指令 — 副团长、策划部、监察部、调度部共用

> 本文件包含核心三部（协调角色）共用的审核流程规则。

---

## 🔄 核心部审核流程

核心三部之间的状态流转遵循以下路径：

```
副团长(Vice) → 策划部(Strategy) → 监察部(Review) → 调度部(Assigned)
                        ↑                    |
                        └────── 驳回退回 ──────┘
```

### 审核原则

1. **策划部**：负责规划拟制，产出可执行方案
2. **监察部**：负责审核把关，确保方案可行且合规
3. **调度部**：负责任务分配和最终汇总验收
4. **副团长**：负责消息分拣和最终回复

### 驳回机制

- 监察部审核不通过 → 退回策划部重新规划（Review → Strategy）
- 调度部复审不通过 → 退回监察部复核（Review → Review）
- 退回时**必须**附带明确的驳回理由和修改要求

### 创建任务权限

只有副团长和策划部可以创建新任务（`create` 命令）。监察部和调度部不创建任务。
